Lightning for Outlook (LFO)

20 minutes read

    ## What is Lightning for Outlook (LFO) - Solution to supercharge the productivity of the sales team. - Sales team can access Salesforce directly from Outlook. - Spend less time switching between apps, logging, and searching — and more time just selling. - Built with a strategic partnership between Salesforce and Microsoft. - LFO 100% cloud-based with all the benefits of the cloud: automatic upgrades and extremely customizable - LFO is available via the Office Store at no additional cost to Sales Cloud users. ### Venn Diagram Outlook and Salesforce

    ## LFO Requirements ### Email Server options - Either Exchange 2016 or 2013 on-premises - Exchange Online with Office 365 ### Email Client App options - Outlook on the web - Microsoft® Outlook® 2016 or 2013 on Windows PC, most recent stable version recommended : see Outlook requirements - Microsoft Outlook for Mac 2016, Mac OS X 10.10 or later with AppleWebKit 600 or later - For the best experience, ensure that Use Cache Mode in the Outlook Account Settings is turned on. ### Browser - Google Chrome most recent stable version - Mozilla Firefox most recent stable version - Apple® Safari® 10 or later, most recent stable version recommended - Microsoft Internet Explorer® 11 (Windows only) - Microsoft Edge, most recent stable version (Windows only) -If you’re using Lightning for Outlook or Inbox Beta for Outlook in a desktop version of Outlook, Microsoft Internet Explorer 11 is required on the desktop computer. However, you’re not required to use Internet Explorer as your default browser.

It’s much like the automatic Salesforce updates you get three times a year.

    ## LFO Limitations ### Shared Mailboxes Currently LFO does not support **Shared Mailboxes**. This limitation is **not** due to a limitation in LFO. It is due to the limitation in the framework based on which LFO is built on. LFO is built on Microsoft's Office Add-in Framework. As per Outlook Add-ins document from Microsoft dated: 06-23-2019 [here](https://docs.microsoft.com/en-us/outlook/add-ins/) : Outlook **does not activate add-ins** if the current message item ( in a compose or read form ) is in a - **shared mailbox** - in another user's mailbox - in an archive mailbox - in a public folder To remove this limitation Salesforce needs this feature to be added to the **Microsoft Office Add-in Framework by Microsoft Team**.
