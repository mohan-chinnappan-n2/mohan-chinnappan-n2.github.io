Use Case

  • In our FSC org, we like to add Near-Realtime Stock Quote Lightning Web Component (LWC)
  • We will using a Custom Settings to store the API key for the Stock Service
  • Apex Class - Server-side Controller will provide this API Key to this LWC Component
  • Stock Rendering will be done via an Util Lightning Web Component
  • Take care of CSP (Content Security Policy) to access the external website
  • Develop this Stock Quote Lightning Web Component using SFDX CLI and VS Code
  • Spin-up a scratch org to do the development
  • All the artifacts are externalized and stored in VCS (git)
  • Test the LWC in the scratch org
  • Push this LWC and related artifacts to a FSC org

How you can push artifacts into a Non-Scratch Orgs

       # ------- login into the Non-Scratch org via CLI
       $ sfdx force:auth:web:login -r https://login|test.salesforce.com 
       # -------- once logged in checked in the org list
       $ sfdx force:org:list
       # -------- you will see this org listed with username
       # -------- source:push to this org
       $ sfdx force:source:push -u username

FSC package install in Scratch Org

  • FSC permission set licenses are available by default in scratch orgs
  • The user needs to install the FSC package using :
  • package:install similar to how they would install any other dependent package.
  $ sfdx force:package:install -h
  Usage: sfdx force:package:install [-w ] [-k ] [-b ] [-r] [-p ] [-s ] [-t ] [-u ] [--json] [--loglevel ] 
  
  install a package in the target org
  
  Flags:
    -k, --installationkey INSTALLATIONKEY  installation key for key-protected package (default: null)
    -r, --noprompt                         allow Remote Site Settings and Content Security Policy websites to send or receive data
                                          without confirmation
    -p, --package PACKAGE                  ID (starts with 04t) or alias of the package version to install
    -b, --publishwait PUBLISHWAIT          number of minutes to wait for subscriber package version ID to become available in the
                                          target org 
    -s, --securitytype SECURITYTYPE        security access type for the installed package (AllUsers,AdminsOnly)
    -u, --targetusername TARGETUSERNAME    username or alias for the target org; overrides default target org
    -t, --upgradetype UPGRADETYPE          the upgrade type for the package installation (Mixed*,DeprecateOnly)
    -w, --wait WAIT                        number of minutes to wait for installation status
    --json                                 format output as json
    --loglevel LOGLEVEL                    logging level for this command invocation (error*,trace,debug,info,warn,fatal)
  
  Supply the ID of the package version to install. The package installs in your default target org unless you supply the username for a different target org.
  
  For package upgrades, to specify options for component deprecation or deletion of removed components, include an --upgradetype value. To delete components that can be safely deleted and deprecate the others, specify --upgradetype Mixed (the default). To deprecate all removed components, specify --upgradetype DeprecateOnly. The default is Mixed.
