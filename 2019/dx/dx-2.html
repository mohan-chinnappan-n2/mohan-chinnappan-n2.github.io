Current State in Most of the Orgs

Challenges with current methods

  • Do not offer Immutable Artifacts
  • Starts with list of components to deployed from one env to another
  • No easy way to aggregating and knowing set-of-metadata moving to different stages
  • From Scratch org deploy Metadata to Sandbox orgs → UAT → PROD
  • Once deployed -- it is happy soup of unpackaged set of metadata

Benefits of Packaging

  • With DX packages we can create Immutable Artifacts on a repeatable manner
  • Version the changes you are making - maintained in the VCS
  • Versioning is built into the packaging - like in Maven
  • Checkout a version of the package - snapshot
  • Make changes - add/edit/remove components
  • Can be used across the environments - transporting
  • Helps to organize metadata in the org

Identifying Metadata that can migrate into a Package

  • Isolate Vertically First
    • Custom Force.com Apps built in-house
    • Extensions of Sales Cloud, Service Cloud, etc
    • Extensions of an AppExchange App
    • Shared Libraries/Functionality
  • Modularize Horizontally as Needed
    • Schema, Business Logic, UI
    • Ownership e.g. Business Unit or Process
    • Shared Libraries/Functionality

General Criteria for Metadata that can migrate into a Package

  • Logical groups of related source/metadata
  • Projects can be tested independently
  • Projects can be released independently
  • Increases Release Flexibility
  • Increases Dev Productivity
