- Overview here
- Create **intent** for anything user might ask - say: Check prices, Book a repair appointment, Operating hours - For each **intent** we provide various ways the user might communicate - For this intent: Book a repair appointment, the might ask in following ways - I'd like to get my equipment fixed this week - My equipment is broken - Can you fix my equipment this week? - Dialog flow uses this info to train a ML model to understand not these examples only, but also numerous other phrases meaning the same/similar intent - Like: My equipment's display unit needs to repaired - So when users says something, our Model will matches the intent which is the good fit for the user ask - For intent, we specify how our app will respond - No, problem, we have an opening this afternoon to this intent: I'd like to get my equipment fixed this week ![df-intent-1](img/df-intent-1.png) ![df-intent-2](img/df-intent-2.png)
- Entity Extraction - Identify the things the user mentions - My **mountain bike** is broken, how I can get it fixed **tomorrow** - These entities are extracted. It has built-in system entities and developer's custom entities - Dates - Places - Amount - Example for bike repair shop - one entity for **@bike-type** - Road Bike - Mountain Bike - Racing Bike - Fixed Gear Bike - another entity for **@service-option** - Tune-up - Repair - Tire change - Upgrade ![df-sys-entity](img/df-sys-entity-1.png) ![df-dev-entity](img/df-dev-entity-0.png) ![df-dev-entity](img/df-dev-entity-1.png) ![df-userentity](img/df-user-entity.png)
- 2 speakers negotiate meaning and understanding of what they say back and forth - that back and forth is called dialog - Linear Dialog - Type of services - Date - Time ![dl-linear](img/df-linear-dl.png) - Non-Linear Dialog - Closer to real conversations ![dl-nlinear](img/df-nlinear-dl.png) - Slotting filling ![dl-slot-filling](img/df-slot-filling-1.png) - Dialog asks for the will for the missing intent ![dl-slot-filling](img/df-slot-filling-2.png)
- Understand with question is being asked - Where in the conversion we are in? - For each intent you can define multiple input and output **contexts** ![context](img/dl-contest-1.png) ![context](img/dl-context-0.png)
