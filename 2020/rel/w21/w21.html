- Do More with the Optimizer App - Analyze Your Lightning Page Performance - Flow features - Salesforce AnyWhere - Interactively Create Engaging Email Templates
- The Optimizer App now has even more capabilities to enhance your Salesforce org’s implementation. - Use two new features to schedule automatic runs and view org metric history and access seven new metrics to improve security. - ![do more with optimizer](img/w21-so-1.gif)
- Discover ways to make your record page perform better based on analysis right inside the Lightning App Builder. - Performance Analysis in App Builder evaluates: - the fields, - instances of the Related Lists component - metadata of a record page. - With the click of a button, get best practices and suggestions for improving page performance and the end-user experience. - ![do more with optimizer](img/w21-perf-analyze.png)
- Trigger flows to run when records are deleted - You no longer need to write Apex code to set this up. - This autolaunched flow runs in the background and updates related records when a record is deleted. - Place Flow Elements Automatically with Auto-Layout (Beta) - makes building flows and aligning elements easier - elements in the canvas are spaced and connected automatically. - No more elements positioned slightly out of alignment! ![flow 1](img/w21-flow-1.gif)

With Salesforce Anywhere, you can: - Stay up to speed by subscribing to real-time alerts based on your preferences - Collaborate in context with real-time chat embedded in your business process - Take action in the moment with suggested actions that are surfaced in context - [Signup Beta](https://www.salesforce.com/form/signup/salesforce-anywhere/)
- Create email templates faster and more easily with visual tools in Email Template Builder. - Instead of adding HTML code, your users can drag elements to create effective email templates. - They can compose email templates with a few clicks to place design elements right where they want them. - ![email builder](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/rn_sales_productivity_ETB_WIP.png)
- You can subscribe to lenses as well as dashboard widgets - Widget subscription is based on a snapshot of your query, a lens subscription is dynamic - First save it in explorer. Then click the Show Subscriptions icon [Subscribe action icon](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/help/images/bi_subscriptions_icon.png) to open the Subscription panel, and click Subscribe. - ![Subscribe](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/help/images/bi_subscription_lens.png) - ![Old](img/ea-lens-1.png) - You can subscribe to any lens that is saved in SAQL format in the explorer. If you must convert a lens to SAQL, we let you know when you try to subscribe. To convert to SAQL, open Query Mode, click Run Query, and then save the lens - ![Run Query](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/help/images/bi_lens_save_as_saql.png)
- Sync Out for Snowflake exports your raw local Salesforce data via Analytics to Snowflake using the Einstein Analytics connector for Snowflake. - Keep your Salesforce data in Snowflake up-to-date using scheduled Data Sync without the need for a third-party ETL tool. - Contact your Salesforce account representative to enable Sync Out for Snowflake. - Turn on Sync Out for Snowflake in Analytics Setup - On the Connect tab of the Data Manager, click Connect to Data. Add a connection on the Output Connections tab and configure it with your **Snowflake account information**. - ![output connections](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/rn_bi_create_output_connection.png) - ![ old](img/ea-op-1.png) - Back on the Connect tab, find the local Salesforce connection with data you want to export to Snowflake. For our example, this local connection includes the account object. Select the down arrow (Down arrow.) next to the connection and click Sync Out. - ![sf synout connect](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/rn_bi_sync_out.png) - Back on the Connect tab, find the local Salesforce connection with data you want to export to Snowflake. For our example, this local connection includes the account object. Select the down arrow (Down arrow.) next to the connection and click Sync Out. - ![sync out](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/rn_bi_sync_out.png) - Sync Out runs as the data sync completes for each local object. For our example, after Data Sync pulls data from the account object into Analytics, Sync Out would then push the same account data to Snowflake.
- Helps to push your prepared data from Analytics into Snowflake when you use Data Prep recipes. - Design powerful Data Prep recipes that combine data from multiple sources, add formula fields, and transform data into datasets tailored to your business needs. - With the EA connector for Snowflake, datasets are written as a table for you without the need to rebuild relationships and business logic. - On the Connect tab of the Data Manager, click Connect to Data. Add a connection on the Output Connections tab and configure it with your Snowflake account information. - ![r sf out](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/rn_bi_connect_snowflake.png) - Build your recipe with Data Prep. In an Output node, select to write to the Output Connection (1). Choose the Snowflake connection to push data to (2) and the Snowflake object that receives the data (3). Map recipe columns to their equivalent external object columns (4). - ![r sf out2](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/rn_bi_recipe_output_node_snowflake.png) - Save the recipe. When the recipe runs, Analytics writes the outcome to Snowflake.
- New intuitive, visual interface that allows you to build recipes with clicks. - Use the new graphical view of a recipe to see at a glance where data comes from, how it flows through the recipe, and where the results are written. - Validate the recipe as you build, preview how raw data is transformed every step of the way. - The new version of Data Prep is called Data Prep and the old version is called Data Prep Classic. - Shows only inputs (source data), appends, aggregates, filters, joins, and the outputs - ![new ui](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/bi_integrate_recipe_large.png) - Each object shown in the graph is called a node. A recipe can have multiple Input, Aggregate, Append, Filter, Join, Transform, and Output nodes. - Although the graph doesn’t show individual data transformations, you can select a Transform node (1) in the graph to see its transformations in the details area (2). - ![transfer node](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/bi_integrate_recipe_transform_details.png) - As in Data Prep Classic, you can select a transformation step in the left panel to preview the results of that transformation in the Preview tab. And you can select any node in the graph to preview the results of that node. - ![ previews](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/bi_integrate_transformation_step_preview.png) - To create a Data Prep recipe, click Create Recipe in the Recipes tab of Data Manager. - ![Create](https://releasenotes.docs.salesforce.com/en-us/winter21/release-notes/release_notes/images/bi_integrate_recipeCreate.png)
